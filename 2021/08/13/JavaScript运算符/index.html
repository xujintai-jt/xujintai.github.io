奇妙的position：absolute！

发表于 2021-08-13 | 更新于: 2021-11-01 | 分类于 css | 次阅读
字数统计: 467字 | 阅读时长 ≈ 1分钟

别小瞧了position：absolute！

一、设置了position：absolute的内联元素的display竟然变成了block！
  • 例1: 行内元素的height、width默认为值auto；当我们直接给行内元素加height，width时，单纯的只是其值发生变化，但在html上并没有实际的渲染。（我们可以通过控制台选中padding时，sapn被全部标出来看出这一点）
  • 例2: 接下来神奇的地方来了。当我们为行内元素span设置了position：absolute时，我们通过控制台可以看到它的dispaly被改变为了block。
- 例3: 我们尝试将span设置为inline-block，通过控制台依旧可以看到它的dispaly被改变为了block。
二、 即使span设置position：absolute变为block，它仍不能像直接设置display:block的span那样成为独占一行‘块元素’(当然，就算给span设置display:block，它也不会成为div那样真正的块元素，这里不展开讨论)。

  • 例1: span设置position：absolute变为block，但不能独占一行。

  • 例2: span设置display:block，独占一行。

  • 例3: 当然，块级元素设置position：absolute后，也变成不能独占一行的block。

三、总结
通过设置position：absolute，元素变为可以设置宽高的block（行为更像是inline-block），但并不具备独占一行的能力。同时，正因为设置了绝对定位的元素行为更像是inline-block（个人理解），所以元素可以设置外边距（margins），且不会与其他边距合并。（块级元素相邻纵向外边距会发生合并）
