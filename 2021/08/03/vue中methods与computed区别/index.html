vue中methods与computed区别

发表于 2021-08-03 | 更新于: 2021-08-03 | 分类于 Vue API | 次阅读
字数统计: 500字 | 阅读时长 ≈ 2分钟

methods不可缓存？computed可缓存？

1.整个页面刷新。结果：computed与methods都调用

2.通过点击“增加num”按钮，改变响应式数据num，并且num在页面上使用。（若响应式数据并未在页面使用，那么即使响应式数据改变，页面也不会发生重新渲染）结果：methods调用，computed不调用。

3.结论 当页面发生重新渲染时，methods会进行重新调用，而computed只在自身依赖的响应式数据发生变化时才会重新调用（整个页面重新刷新除外）。另外，只有页面使用了响应式数据，并且响应式数据发生变化时页面才会重新渲染。
实际上每一个在template上使用的响应式数据都是一个watcher，当数据发生变化时，相应的dep会notify相关的wather进行页面更新操作，比如收集num相关watcher的dep会通知所有num相关的watcher进行update。
